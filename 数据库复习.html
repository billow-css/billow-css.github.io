                                                            3-SQL基础 （基于SQL_server）

                                                            SQL的组成

                                                            • DDL数据定义语言（Data Definition Language）：定义数据库的结构，包括定义表、索引等；

                                                            • DML数据操纵语言（Data Manipulation Language）：对数据库进行查询、插入与删除操作；

                                                            • DCL数据控制语言（Data Control Language）：数据库的安全性控制和完整性控制；

                                                            SQL语句分类

                                                            • 数据定义：创建，更新，撤销模式与对象；

                                                            • 数据查询：对数据进行查询；

                                                            • 数据操纵：完成数据库的数据更新

                                                            • 数据控制。

                                                            SELECT语句用法

                                                            整体框架

                                                            xxxxxxxxxx
                                                            SELECT [ALL | DISTINCT] [func(]<target key 1>[)][AS <name> <varname>] [<target key n>]
                                                            FROM <target sheet 1> [<target sheet n>][[LEFT|RIGHT|INNER] JOIN <target sheet m> ON <condition>]
                                                            WHERE <condition>
                                                            

                                                            GROUP BY <key>
                                                            HAVING <condition>
                                                            ORDER BY <key> [ASC|DESC]

                                                            笔记

                                                            • SELECT语句顺序：

                                                              x
                                                              [SELECT代码块顺序如下：]
                                                              ├── FROM
                                                              ├── WHERE
                                                              ├── GROUP BY
                                                              ├── HAVING
                                                              ├── SELECT
                                                              └── ORDER BY

                                                            • 条件表达式的具体写法：

                                                            • 如果需要模糊查询，将肯能需要模糊的地方使用_或者%表示

                                                            • func()聚合函数一般情况下需要配合GROUP BY函数使用

                                                              请注意：如果出现聚合函数，那么SELECT中只允许出现聚合函数和分组列。

                                                            • 连接查询

                                                              针对WHERE和JOIN主导的连接查询：JOIN 是语法层面的表连接，WHERE 是筛选层面的连接；JOIN 更清晰、更安全、区分内外连接、性能更好、可规避笛卡尔积；且JION可以保留不匹配与空行，这是WHERE做不到的。

                                                              • [INNER] JOIN 普通 JOIN：只留两边都匹配的数据

                                                              • LEFT JOIN 保留左表全部，右表匹配不到填 NULL

                                                              • RIGHT JOIN 保留右表全部，左表匹配不到填 NULL

                                                            • 子查询

                                                              一般在WHERE处使用确定集合或者比较运算来子查询

                                                              xxxxxxxxxx
                                                              WHERE <key> {IN|NOT IN|[<Arithmetic Comp> {ALL|SOME|ANY}} (
                                                                SELECT <key> FROM <sheet>
                                                                WHERE <condition>
                                                              )

                                                              • 对于ALL SOME ANY: ALL ~ ∀ ANY ~ ∃

                                                              • EXISTS 不需要外层字段比较，只判断子查询是否存在结果，返回布尔值； 但子查询内部必须写关联条件

                                                            • 集合查询

                                                              在两个查询子句中，使用UNION INTERSECT和EXCEPT对两者进行拼接。

                                                            1-数据库绪论

                                                            基本概念

                                                            • DB 数据库：长期存储在计算机内、有组织、可共享的大量数据集合

                                                            • DBMS 数据库管理系统：管理数据库的系统软件（你前面 SQL 里的 DDL/DML/DCL 就是它提供的）

                                                            • DBS 数据库系统：DB+DBMS + 应用 + 用户，整体系统

                                                            • DBA 数据库管理员

                                                            特点

                                                            1. 数据结构化

                                                            2. 数据共享性高、冗余度低，易于扩充

                                                            3. 数据独立性高（逻辑 + 物理）

                                                            4. 数据统一管理控制（安全性、完整性、并发、故障恢复）

                                                            数据管理技术 3 个阶段

                                                            人工管理阶段 → 文件系统阶段 → 数据库系统阶段

                                                            联系的三种类型

                                                            • 一对一 1:1

                                                            • 一对多 1:n

                                                            • 多对多 m:n

                                                            数据库的三级模式和二级映像

                                                            三级模式

                                                            • 模式：整个数据库的全局逻辑视图，所有用户的公共数据视图

                                                            • 外模式：单个用户看到的局部视图，是模式的子集

                                                            • 内模式：数据库物理存储结构，描述数据怎么存在硬盘上

                                                            二级映像

                                                            外模式 / 模式映像 → 逻辑独立性

                                                            模式变了（表结构改了），修改该映像，用户看到的外模式不变，应用程序不用改

                                                            模式 / 内模式映像 → 物理独立性

                                                            内模式变了（换存储位置、建索引），修改该映像，模式不变，应用程序不用改

                                                            数据模型

                                                            概念数据模型（Conceptual Data Model）

                                                            • 面向谁：用户、业务人员、设计人员

                                                            • 作用：描述现实世界的实体和关系，完全不考虑数据库、不考虑怎么存

                                                            • 代表：E‑R 图（实体‑联系图）（详见CH4）

                                                            • 特点：独立于数据库系统、独立于存储

                                                            ER图

                                                            1. 三要素：实体，属性，联系

                                                            2. 实体型，实体值，实体集。

                                                            逻辑数据模型（Logical Data Model）

                                                            • 面向谁：数据库开发人员

                                                            • 作用：把 E‑R 图转成数据库表结构，描述数据之间的逻辑关系

                                                            • 代表：关系模型（表、主键、外键、约束），就是的内容

                                                            • 对应三级模式：模式 + 外模式

                                                            数据模型三要素：数据结构，数据操作，数据完整性约束

                                                            物理数据模型（Physical Data Model）

                                                            • 面向谁：DBA、底层存储

                                                            • 作用：描述数据在磁盘上怎么存、索引怎么建、数据块怎么放

                                                            • 代表：存储结构、索引、聚簇、文件组织

                                                            • 对应三级模式：内模式

                                                            2-关系数据库基础

                                                            形式化定义

                                                            • 域：一组具有相同数据类型的值的集合，是属性的取值范围。

                                                            • 笛卡尔积：给定一组域，所有域的所有取值的组合。

                                                            • 关系：笛卡尔积的有限子集，一张二维表。

                                                              • 元组：关系中的一行（记录）

                                                              • 属性：关系中的一列（字段）

                                                              • 超码：能唯一标识一个元组（一行）的属性集合。

                                                              • 主码（主键）：唯一标识一个元组的属性 / 属性组

                                                              • 外码（外键）：引用另一关系主码的属性，实现参照完整性

                                                              • 候选码：能唯一标识元组的最小属性集；主码从候选码中选定

                                                              • 主属性：候选码中的属性；非主属性：不在任何候选码中的属性

                                                              关系模式

                                                              (1)R(U,D,dom,F)

                                                              • U属性集合

                                                              • D全部域

                                                              • dom属性和域的映像集合

                                                              • F属性间依赖关系的集合

                                                            数据完整性

                                                            1. 实体完整性：主码非空、唯一，不能取空值

                                                            2. 参照完整性：外码取值为空，或等于被参照关系的主码值

                                                            3. 用户定义完整性：NOT NULL、UNIQUE、CHECK、DEFAULT 等业务约束

                                                            关系代数

                                                            传统集合运算（并、交、差、笛卡尔积）

                                                            • 并 ∪：合并两个关系，去重

                                                            • 交 ∩：取同时存在的元组

                                                            • 差 −：属于 R 但不属于 S 的元组

                                                            • 笛卡尔积 ×：所有元组两两拼接

                                                            5-关系规范化理论

                                                            函数依赖

                                                            • 函数依赖：X→Y，若 X 属性值确定，则 Y 属性值唯一确定

                                                            • 完全函数依赖：X 的全部属性才能决定 Y

                                                            • 部分函数依赖：X 的部分属性即可决定 Y

                                                            • 传递函数依赖：X→Y, Y→Z，且 Y 不决定 X、Z∉Y

                                                            范式

                                                            1. 1NF 第一范式：属性不可再分，原子性

                                                            2. 2NF 第二范式：满足 1NF，消除部分函数依赖，非主属性完全依赖主码

                                                            3. 3NF 第三范式：满足 2NF，消除传递函数依赖，非主属性不传递依赖主码

                                                            4. BCNF 范式：满足 3NF，主属性不依赖于非主属性，消除主属性对主码的部分 / 传递依赖

                                                            数据依赖公理系统

                                                            Armstrong公理系统

                                                            1. 自反律：子集依赖于超集，Y⊆X⇒X→Y

                                                            2. 增广律：X→Y⇒XZ→YZ

                                                            3. 传递律：X→Y,Y→Z⇒X→Z

                                                            推论

                                                            • 合并：X→Y,X→Z⇒X→YZ

                                                            • 分解：X→YZ⇒X→Y,X→Z

                                                            • 伪传递：X→Y,YZ→W⇒XZ→W

                                                            函数依赖集闭包

                                                            给定函数依赖集 F，利用阿姆斯特朗公理能推导出的所有函数依赖的全体，就叫 F 的闭包，记作 F+。

                                                            给定属性集 X，在 F 下，由 X 能推导出的所有属性的集合，记作 X+。
